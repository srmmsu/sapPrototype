PROJECT CONTEXT (please read before responding)

I am developing a Secure Access Portal (SAP) licensing system with the following architecture:

Backend:

* Django backend running on Ubuntu
* API endpoints for fingerprint registration and verification
* Admin panel to set:

  * offline credit
  * license expiry date

Frontend:

* React (Create React App)
* PWA behaviour using service workers
* deployed locally with:
  serve -s build -l 3000

Core Features:

1. User registers device using fingerprint (FingerprintJS)
2. Backend verifies user and sends:

   * offline_credit
   * expiry_date
3. Frontend stores:

   * userId
   * fingerprint
   * offlineCredit
   * licenseExpiry
     in localStorage
4. When backend is unreachable:

   * offline login allowed
   * credit decremented locally
5. Only one offline session allowed per device
6. License expiry should block login even offline

Testing setup:

* Ubuntu laptop acting as server (IP like 192.168.1.x)
* Clients tested on:

  * Ubuntu Chrome
  * Android Chrome
  * iPad Chrome

Problem encountered:
When backend server goes down, SAP sometimes shows blank screen or incorrect behaviour. The project became complex due to PWA, service workers, and React render logic interacting.

Decision:
We decided to temporarily pause the main project and create a SMALL PARALLEL PROTOTYPE to understand each component clearly.

Prototype goals:
Stage 1: Minimal React PWA shell that works offline
Stage 2: Backend ping detection (online/offline indicator)
Stage 3: Local offline credit decrement
